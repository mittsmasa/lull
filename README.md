# Lull（ラル）

---

## 1. ブランド・アイデンティティ (Brand Identity)

### 1.1 ネーミングの定義

- **名称**: Lull（ラル）
- **語源**:
  - 英語で「凪（なぎ）」「静まり」「一時的な休止」を意味する。
  - Lullaby（子守唄）の語源。
- **コンセプト**:
  - 喧騒を離れ、カノンハウスの椅子に深く腰掛け、最初の音が鳴る瞬間の「贅沢な静寂」を象徴。
  - 単なる「管理」ではなく、演奏が始まる前の「静かな高揚感」をデザインする。

### 1.2 アプリ概要

Lull は、ピアノ発表会などの小規模コンサート・リサイタルを運営するための Web アプリケーション。イベントの企画から当日の受付・進行管理までをカバーする。

ロールは **主催者（Organizer）・出演者（Performer）・ゲスト（Guest）** の3つ。機能要件・ロール定義・権限の詳細は [要件定義書](docs/requirements.md) を参照。

## 2. キーワード・世界観 (Brand World)

Lull のデザインと体験は、3つのキーワードに導かれる。

### ハレの日

出演者にとっても、ゲストにとっても、発表会は「ハレの日」。日常の練習が実を結ぶ舞台であり、大切な人とそれを分かち合う時間。Lull はその一日を、招待の瞬間から丁寧に届ける。

### 力みのない本気

出演者は本気で演奏する。でもそれを受け取る場は、肩の力が抜けている。構えず、でも手は抜かない。懇親会で酒を酌み交わす空気感のまま、研鑽の成果を味わう。そのバランスが Lull。

### 余韻

演奏が終わった後も残り続ける響き。Lull は発表会の「当日だけ」を切り取らない。招待を受け取った瞬間の期待感から、終演後の記憶まで ― 体験を前後に広げ、自然に満たす。

## 3. UI / デザイン指針 (Design Direction)

### 3.1 原風景

Lull のデザインは **「招待状」** を原風景にする。紙の質感、インクの色、余白の取り方 ― 受け取った人が「丁寧に作られたものだ」と感じる、あの手触り。画面の中に、紙と光と静けさを再現する。

デザインシステムの詳細は [Lull Design System](https://v0-design-aesthetic-guidance.vercel.app/design-system) を参照。以下は主要な指針の抜粋。

### 3.2 カラーパレット

| 名前 | CSS 変数 | 役割 | イメージ |
| --- | --- | --- | --- |
| 生成り（きなり） | `--background` | ベースカラー | 温度のある白。自然光の中の紙 |
| テラコッタ | `--primary` | アクセント | 封蝋の赤み。ハレの日の華やぎ（装飾的に限定使用） |
| 淡温色 | `--secondary` | セカンダリ | 温かみのあるベージュ |
| キャメル | `--accent` | サブカラー | 革の手触り。温かみのある中間色 |
| 墨色 | `--foreground` | テキスト | 万年筆のインク。深みと柔らかさを両立 |

その他の機能色: `--card`, `--muted`, `--border`, `--input`, `--ring`, `--destructive`

彩度の高い原色は使わない。色は語りかけるのではなく、寄り添う。

### 3.3 タイポグラフィ

- **和文**: Shippori Mincho ― 横線の細さと縦線の強さが「力みのない本気」を体現する
- **英文**: Cormorant Garamond ― 和文明朝と通底する品格。ロゴタイプとしても機能する
- **組み方**: 行間 1.6 以上（`leading-relaxed`）、見出しは `tracking-wide` 以上

**Type Scale**:

| 用途 | サイズ |
| --- | --- |
| Display | 4.5rem |
| H1 | 3rem |
| H2 | 1.875rem |
| H3 | 1.25rem |
| Body | 1rem |
| Small | 0.875rem |
| Caption | 0.75rem |

### 3.4 スペーシング

| トークン | 値 | Tailwind |
| --- | --- | --- |
| xs | 4px | `gap-1` / `p-1` |
| sm | 8px | `gap-2` / `p-2` |
| md | 16px | `gap-4` / `p-4` |
| lg | 24px | `gap-6` / `p-6` |
| xl | 32px | `gap-8` / `p-8` |
| 2xl | 48px | `gap-12` / `p-12` |
| 3xl | 64px | `gap-16` / `p-16` |
| section | 128px | `gap-32` / `py-32` |

ボーダー半径は `rounded-sm` を基本とし、過度な丸みを避ける。

### 3.5 空間設計

- **間（ま）を活かす** ― 余白はデザインの一部。ただし余白のための余白にしない。情報にたどり着くまでの距離が適切であること
- **密度より呼吸、ただし情報は届ける** ― 一画面に詰め込まないが、必要な情報は迷わず見つかる。情報量を意図的に抑え、スクロール・ページ遷移を許容する
- **非対称の静けさ** ― 左右対称にこだわらず、自然光が片側から差し込むような重心のある構図

### 3.6 モーション

- **すっと現れる** ― 0.2〜0.4秒、`ease-out` のイージング。フェードイン + Y軸微細移動の組み合わせ
- **余韻を持たせる** ― 即座に現れず、消えるときも名残を残す
- **控えめ、でも不在ではない** ― ホバー時スケールは 1.02 以下。気づかないくらいがちょうどいい
