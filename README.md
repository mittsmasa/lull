# Lull（ラル）

## ― 奏の特等席へ、ようこそ ―

---

## 1. ブランド・アイデンティティ (Brand Identity)

### 1.1 ネーミングの定義

- **名称**: Lull（ラル）
- **語源**:
  - 英語で「凪（なぎ）」「静まり」「一時的な休止」を意味する。
  - Lullaby（子守唄）の語源。
- **コンセプト**:
  - 喧騒を離れ、カノンハウスの椅子に深く腰掛け、最初の音が鳴る瞬間の「贅沢な静寂」を象徴。
  - 単なる「管理」ではなく、演奏が始まる前の「静かな高揚感」をデザインする。

### 1.2 アプリ概要

Lull は、ピアノ発表会における **招待・座席・当日体験** を一つに繋ぐアプリケーション。

管理者が座席プールを設定し、出演者がその枠内でゲストに招待を送り、ゲスト情報を管理する。当日はプログラムや進行をリアルタイムに共有し、会場にいる全員が同じ流れを見渡せる。

**3つのロール**:

- **管理者** ― 座席プールの設定、全体の運営管理
- **出演者** ― ゲストへの招待送付、自身のゲスト管理
- **ゲスト** ― 招待の受領、当日のプログラム閲覧

## 2. キーワード・世界観 (Brand World)

Lull のデザインと体験は、3つのキーワードに導かれる。

### ハレの日

出演者にとっても、ゲストにとっても、発表会は「ハレの日」。日常の練習が実を結ぶ舞台であり、大切な人とそれを分かち合う時間。Lull はその一日を、招待の瞬間から丁寧に届ける。

### 力みのない本気

出演者は本気で演奏する。でもそれを受け取る場は、肩の力が抜けている。構えず、でも手は抜かない。懇親会で酒を酌み交わす空気感のまま、研鑽の成果を味わう。そのバランスが Lull。

### 余韻

演奏が終わった後も残り続ける響き。Lull は発表会の「当日だけ」を切り取らない。招待を受け取った瞬間の期待感から、終演後の記憶まで ― 体験を前後に広げ、自然に満たす。

## 3. UI / デザイン指針 (Design Direction)

### 3.1 原風景

Lull のデザインは **「招待状」** を原風景にする。紙の質感、インクの色、余白の取り方 ― 受け取った人が「丁寧に作られたものだ」と感じる、あの手触り。画面の中に、紙と光と静けさを再現する。

### 3.2 カラーパレット

| 名前 | 役割 | イメージ |
| --- | --- | --- |
| 生成り（きなり） | ベースカラー | 温度のある白。自然光の中の紙 |
| テラコッタ | アクセント | 封蝋の赤み。ハレの日の華やぎ |
| キャメル | サブカラー | 革の手触り。温かみのある中間色 |
| 墨色 | テキスト | 万年筆のインク。深みと柔らかさを両立 |

彩度の高い原色は使わない。色は語りかけるのではなく、寄り添う。

### 3.3 タイポグラフィ

- **和文**: 明朝系 ― 横線の細さと縦線の強さが「力みのない本気」を体現する
- **英文**: セリフ系 ― 和文明朝と通底する品格
- **組み方**: ゆったりとした字間と行間。文字が呼吸できる余白を持たせる

### 3.4 空間設計

- **間（ま）を活かす** ― 余白はデザインの一部。ただし余白のための余白にしない。情報にたどり着くまでの距離が適切であること
- **密度より呼吸、ただし情報は届ける** ― 一画面に詰め込まないが、必要な情報は迷わず見つかる
- **非対称の静けさ** ― 左右対称にこだわらず、自然光が片側から差し込むような重心のある構図

### 3.5 モーション

- **すっと現れる** ― 0.2〜0.4秒の穏やかなイージング。要素が空間に自然に馴染む
- **余韻を持たせる** ― 即座に現れず、消えるときも名残を残す
- **控えめ、でも不在ではない** ― 気づかないくらいがちょうどいい

## 4. ユーザーロール (User Roles)

Lull には3つのロールがある。管理者は出演者を兼ねる。

### 4.1 管理者 (Admin)

出演者の中から数名がアサインされる。アプリに対するほぼすべての権限を持つ。

- 座席プールの設定と管理
- 出演者ごとの招待枠（アロットメント）の割り当て
- 全体の受付状況・来場状況の把握
- プログラム・当日進行の編集
- 出演者の管理
- 当日の参加チェック（ゲストの招待情報を確認し、来場を記録）

### 4.2 出演者 (Performer)

有志の演奏者。自分のゲストを招き、迎え入れるホスト。

- ゲストへの招待送付
- ゲスト情報の管理（受諾・辞退・来場の確認）
- 当日のプログラム・進行の閲覧
- 当日の参加チェック（自分のゲストの来場を確認・記録）

### 4.3 ゲスト (Guest)

出演者から招待を受け、発表会に訪れる人。

- 招待の受領（受諾 / 辞退）
- 当日のプログラム閲覧
- 来場時に招待情報（QRコード等）を提示

## 5. 技術方針 (Tech Stack)

### 5.1 アーキテクチャ

Web アプリケーション（PWA）。URL で共有でき、ゲストはインストール不要でアクセスできる。

### 5.2 フロントエンド

- **フレームワーク**: Next.js
- **言語**: TypeScript
- **パッケージマネージャ**: pnpm

### 5.3 API

- **Hono RPC** を Next.js Route Handler に配置
- クライアント ↔ サーバー間の型安全な通信を実現

### 5.4 データベース

- **Cloudflare D1**（SQLite ベース）
- **Drizzle ORM** でスキーマ定義・マイグレーション・クエリを型安全に管理

### 5.5 認証

- ソーシャルログイン（Google / LINE 等の OAuth）
- 管理者・出演者: ログイン必須
- ゲスト: 招待リンクからのアクセス（ログイン不要を想定）

### 5.6 PWA

- **Serwist** で Service Worker を管理
- プログラム閲覧等のオフライン対応

### 5.7 インフラ

- **Cloudflare Pages**: フロントエンドのホスティング
- **Cloudflare Workers / Pages Functions**: サーバーサイド処理
- **Cloudflare D1**: データベース
- `@opennextjs/cloudflare` で Next.js を Cloudflare にデプロイ
